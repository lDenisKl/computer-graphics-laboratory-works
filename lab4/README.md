# Лабораторная работа 4 — Алгоритмы отсечения отрезка

## Постановка задачи

Дан отрезок с концами **P₁** и **P₂**. Требуется найти видимую часть отрезка, то есть ту его часть, которая лежит внутри заданного окна отсечения. Задача решается тремя различными алгоритмами и визуализируется на одном графике.

## Входные данные (`data.json`)

```json
{
  "polygon":   [[-80,-60], [100,-80], [150,20], [60,100], [-60,80]],
  "rectangle": { "xmin": -120, "ymin": -100, "xmax": 120, "ymax": 100 },
  "segment":   { "p1": [-200,-150], "p2": [200,160] }
}
```

| Поле        | Описание                                                        |
|-------------|-----------------------------------------------------------------|
| `polygon`   | Вершины выпуклого многоугольника — окно отсечения для алгоритма Сайрус–Бек |
| `rectangle` | Ось-параллельный прямоугольник — окно для Коэн–Сазерленд и метода средней точки |
| `segment`   | Концы отрезка P₁ и P₂                                          |

## Алгоритм 1 — Сайрус–Бек

### Идея

Алгоритм работает с **произвольным выпуклым многоугольником**. Отрезок параметризуется:

```
P(t) = P₁ + t·D,   D = P₂ − P₁,   t ∈ [0, 1]
```

Для каждого ребра многоугольника вычисляется параметр `t` пересечения прямой отрезка с прямой ребра, а затем определяется тип пересечения: **потенциально входящее (PE)** или **потенциально выходящее (PL)**.

### Условие ориентации

Алгоритм требует, чтобы вершины многоугольника были заданы **против часовой стрелки (CCW)**. Функция `ensure_ccw` проверяет знак площади через формулу Гаусса и при необходимости разворачивает список вершин.

### Вычисление параметра `t`

Для ребра `A→B` внутренняя нормаль:

```
Nᵢ = (−(B.y − A.y),  B.x − A.x)
```

Параметр пересечения:

```
t = dot(Nᵢ, A − P₁) / dot(Nᵢ, D)
```

- Если `dot(Nᵢ, D) < 0` — прямая отрезка **входит** в полуплоскость ребра → **PL**-пересечение, обновляем `tL = min(tL, t)`.
- Если `dot(Nᵢ, D) > 0` — прямая отрезка **выходит** → **PE**-пересечение, обновляем `tE = max(tE, t)`.
- Если `dot(Nᵢ, D) = 0` — отрезок параллелен ребру. Если при этом `dot(Nᵢ, A − P₁) < 0` — отрезок целиком снаружи, возвращаем `None`.

### Результат

После обхода всех рёбер:

- Если `tE > tL` — отрезок полностью вне окна.
- Иначе:
  - **Q₁** = P₁ + tE · D — точка входа в окно (наибольший PE-параметр).
  - **Q₂** = P₁ + tL · D — точка выхода из окна (наименьший PL-параметр).

### На графике

| Элемент | Описание |
|---------|----------|
| Синие штриховые линии | Прямые, продолжающие каждое ребро многоугольника |
| Закрашенный многоугольник | Само окно отсечения |
| Красные точки PE₁…PEₙ | Все потенциальные точки входа (по одной на каждое ребро) |
| Оранжевые точки PL₁…PLₙ | Все потенциальные точки выхода |
| Зелёные ромбы Q₁, Q₂ | Итоговые точки отсечения (победившие PE и PL) |

## Алгоритм 2 — Коэн–Сазерленд

### Идея

Алгоритм работает с **прямоугольником**, стороны которого параллельны осям координат. Плоскость делится на **9 регионов** двумя вертикальными (x = xmin, x = xmax) и двумя горизонтальными (y = ymin, y = ymax) линиями. Каждому региону присваивается **4-битный код**.

### Коды регионов

```
бит 0 (LEFT)   — x < xmin
бит 1 (RIGHT)  — x > xmax
бит 2 (BOTTOM) — y < ymin
бит 3 (TOP)    — y > ymax
```

Если все биты нулевые — точка внутри окна.

### Шаги алгоритма

1. Вычислить коды `c1` и `c2` для концов отрезка.
2. **Тривиальное принятие**: если `c1 | c2 == 0` — оба конца внутри, отрезок видим целиком.
3. **Тривиальное отклонение**: если `c1 & c2 != 0` — оба конца в одном внешнем регионе, отрезок невидим.
4. Иначе — взять конец с ненулевым кодом, найти пересечение с соответствующей границей (TOP → y = ymax, BOTTOM → y = ymin, RIGHT → x = xmax, LEFT → x = xmin), заменить этот конец найденной точкой и повторить.

### Результат

Алгоритм итеративно «срезает» отрезок по одной границе за шаг, пока не наступит тривиальное принятие или отклонение. Финальные координаты концов — это **Q₁** и **Q₂**.

### На графике

| Элемент | Описание |
|---------|----------|
| Синие штриховые линии | Четыре граничные прямые (x = xmin/xmax, y = ymin/ymax), делящие плоскость на 9 регионов |
| Закрашенный прямоугольник | Само окно отсечения |
| P₁', P₂' | Точки пересечения отрезка со сторонами прямоугольника |
| Зелёные ромбы Q₁, Q₂ | Итоговые точки отсечения |

## Алгоритм 3 — Метод средней точки

### Идея

Алгоритм работает с **прямоугольником** и использует **бинарное деление** (бисекцию) отрезка для нахождения точек пересечения с границей. Не требует аналитического вычисления пересечений — работает численно.

### Шаги алгоритма

1. Если оба конца внутри — отрезок видим целиком.
2. Если коды регионов обоих концов имеют общий бит (`c1 & c2 != 0`) — отрезок невидим.
3. Иначе для каждого конца, лежащего снаружи, выполняется **бисекция**:
   - Берётся пара (внешняя точка, внутренняя точка).
   - Вычисляется средняя точка.
   - Если средняя точка внутри — она становится новой внутренней; иначе — новой внешней.
   - Повторяется `iters = 52` раз (достаточно для точности числа с плавающей запятой `float64`).

### Результат

После бисекции получаем координаты граничных точек **Q₁** и **Q₂** с машинной точностью.

### На графике

Аналогично Коэн–Сазерленд, поскольку окно одинаковое. Разница только в способе нахождения Q₁ и Q₂ — численным, а не аналитическим методом. Результаты совпадают в пределах погрешности `float64`.

## Разбор кода (`task4.py`)

### Загрузка данных

```python
with open('data.json', encoding='utf-8') as f:
    d = json.load(f)

polygon = [tuple(p) for p in d['polygon']]
P1 = tuple(d['segment']['p1'])
P2 = tuple(d['segment']['p2'])
r = d['rectangle']
xmin, ymin, xmax, ymax = r['xmin'], r['ymin'], r['xmax'], r['ymax']
```

JSON возвращает вершины как списки `[x, y]`. Функции алгоритмов принимают кортежи `(x, y)` — поэтому каждый элемент оборачивается в `tuple`. Прямоугольник разбирается в четыре отдельных имени, чтобы в алгоритмах не писать `r['xmin']` каждый раз.

---

### `dot(a, b)` — скалярное произведение

```python
def dot(a, b): return a[0]*b[0] + a[1]*b[1]
```

Используется везде, где нужно вычислить `dot(N, D)` (проекция направления отрезка на нормаль ребра) или `dot(N, A−P₁)` (знак показывает, с какой стороны плоскости находится точка). Вынесена отдельно, чтобы не дублировать выражение в каждой функции.

---

### `ensure_ccw(poly)` — ориентация многоугольника

```python
def ensure_ccw(poly):
    area = sum(poly[i][0]*poly[(i+1)%len(poly)][1] - poly[(i+1)%len(poly)][0]*poly[i][1]
               for i in range(len(poly)))
    return poly[::-1] if area < 0 else poly
```

Формула Гаусса (формула трапеций) вычисляет удвоенную знаковую площадь многоугольника. Если результат отрицательный — вершины идут **по часовой стрелке (CW)**, и список разворачивается (`[::-1]`). Алгоритм Сайрус–Бек работает правильно только при обходе **CCW**, потому что тогда внутренняя нормаль к ребру `A→B` равна `(−dy, dx)` и указывает *внутрь* многоугольника.

---

### `rect_intersections(p1, p2)` — точки пересечения со сторонами прямоугольника

```python
def rect_intersections(p1, p2):
    D = (p2[0]-p1[0], p2[1]-p1[1])          # вектор направления отрезка P1→P2
    pts = []
    for a, b in [((xmin,ymin),(xmax,ymin)),  # нижняя сторона
                 ((xmax,ymin),(xmax,ymax)),  # правая сторона
                 ((xmax,ymax),(xmin,ymax)),  # верхняя сторона
                 ((xmin,ymax),(xmin,ymin))]: # левая сторона
        N   = (-(b[1]-a[1]), b[0]-a[0])     # нормаль к ребру a→b
        den = dot(N, D)                      # проекция D на нормаль
        if den == 0: continue                # D параллелен ребру → нет пересечения
        t = dot(N, (a[0]-p1[0], a[1]-p1[1])) / den  # параметр пересечения
        p = (p1[0]+t*D[0], p1[1]+t*D[1])   # координата точки P1 + t·D
        ex, ey = sorted([a[0],b[0]]), sorted([a[1],b[1]])  # bbox ребра
        if ex[0]-1e-9 <= p[0] <= ex[1]+1e-9 \
        and ey[0]-1e-9 <= p[1] <= ey[1]+1e-9 \
        and 0 <= t <= 1:                     # точка лежит на ребре И на отрезке
            pts.append(p)
    return pts
```

#### Переменные по одной

| Переменная | Что хранит | Как вычисляется |
|------------|-----------|----------------|
| `D` | Вектор направления отрезка P1→P2 | `(p2.x − p1.x, p2.y − p1.y)` |
| `a, b` | Пара вершин одного ребра прямоугольника | Берётся из фиксированного списка четырёх сторон |
| `N` | Нормаль к ребру `a→b` (перпендикуляр, смотрит внутрь) | `(−(b.y − a.y), b.x − a.x)` — стандартный поворот вектора на 90° влево |
| `den` | Проекция вектора `D` на нормаль ребра | `dot(N, D)` |
| `t` | Параметр точки пересечения прямой отрезка с прямой ребра | `dot(N, a − p1) / den` — из уравнения плоскости |
| `p` | Координаты самой точки пересечения | `p1 + t · D` |
| `ex, ey` | Диапазоны координат ребра (его bounding box) | `sorted([a.x, b.x])` и `sorted([a.y, b.y])` |

#### Почему именно такая формула для `t`?

Точка лежит на прямой ребра, когда она находится точно на плоскости, заданной нормалью `N` и точкой `a`.
Уравнение плоскости: `dot(N, P − a) = 0`.
Подставляем параметрическую точку отрезка `P = p1 + t·D`:

```
dot(N, p1 + t·D − a) = 0
dot(N, p1 − a) + t · dot(N, D) = 0
t = dot(N, a − p1) / dot(N, D)
```

#### Зачем проверка `ex[0] <= p.x <= ex[1]` и `0 <= t <= 1`?

`t` вычисляется для **бесконечной прямой** отрезка. Нужны два ограничения:
- `ex/ey`-проверка: точка лежит не просто на прямой ребра, а на **конечном ребре** (иначе угол прямоугольника дал бы лишние точки).
- `0 <= t <= 1`: точка лежит на самом **отрезке** P1P2, а не на его продолжении.

Допуск `1e-9` нужен, потому что из-за округления `float` координата угловой точки может оказаться чуть за пределами `[a, b]`.

> **Важно:** эта функция не участвует в отсечении — она вызывается только при рисовании промежуточных точек P₁', P₂' на панелях Коэн–Сазерленд и средней точки.

### `cyrus_beck(p1, p2, poly)` — алгоритм Сайрус–Бек

```python
def cyrus_beck(p1, p2, poly):
    poly = ensure_ccw(poly)                    # вершины должны идти CCW
    D = (p2[0]-p1[0], p2[1]-p1[1])            # вектор направления P1→P2
    tE, tL = 0.0, 1.0                          # границы видимого интервала [tE, tL]
    pe, pl = [], []                             # точки пересечений для графика
    for i in range(len(poly)):
        A, B = poly[i], poly[(i+1)%len(poly)]  # текущее ребро многоугольника
        N = (-(B[1]-A[1]), B[0]-A[0])          # внутренняя нормаль ребра A→B
        den = dot(N, D)                         # «скорость» входа/выхода
        if den == 0:                            # D параллелен ребру
            if dot(N, (A[0]-p1[0], A[1]-p1[1])) < 0:
                return None                    # P1 снаружи этого ребра → весь отрезок мимо
            continue
        t = dot(N, (A[0]-p1[0], A[1]-p1[1])) / den  # параметр пересечения
        p = (p1[0]+t*D[0], p1[1]+t*D[1])             # сама точка пересечения
        if den < 0:                             # выходим из полуплоскости этого ребра
            pl.append(p); tL = min(tL, t)      # PL: сужаем правую границу
        else:                                   # входим в полуплоскость этого ребра
            pe.append(p); tE = max(tE, t)      # PE: сужаем левую границу
    if tE > tL: return None                    # интервал схлопнулся → отрезок вне окна
    return (p1[0]+tE*D[0], p1[1]+tE*D[1]), \
           (p1[0]+tL*D[0], p1[1]+tL*D[1]), pe, pl
```

#### Переменные по одной

| Переменная | Что хранит | Начальное значение | Как меняется |
|------------|-----------|-------------------|-------------|
| `D` | Вектор-направление отрезка P1→P2 | — | Не меняется |
| `tE` | «Earliest entry» — наибольший параметр из всех PE-пересечений | `0.0` (левый конец P1) | `max(tE, t)` при каждом PE |
| `tL` | «Latest leaving» — наименьший параметр из всех PL-пересечений | `1.0` (правый конец P2) | `min(tL, t)` при каждом PL |
| `pe` | Список всех точек PE (для графика) | `[]` | `append(p)` при каждом PE |
| `pl` | Список всех точек PL (для графика) | `[]` | `append(p)` при каждом PL |
| `A, B` | Начало и конец текущего ребра | — | Меняются в каждой итерации |
| `N` | Внутренняя нормаль ребра `A→B` | — | Пересчитывается для каждого ребра |
| `den` | `dot(N, D)` — проекция направления отрезка на нормаль | — | Пересчитывается для каждого ребра |
| `t` | Параметр пересечения прямой отрезка с прямой ребра | — | Новый для каждого ребра |
| `p` | Координаты точки пересечения | — | Новая для каждого ребра |

#### Откуда берётся нормаль `N = (−dy, dx)`?

Ребро идёт из `A` в `B`. Вектор ребра — `(dx, dy) = (B.x−A.x, B.y−A.y)`. Поворот на 90° **влево** (против часовой стрелки): `(−dy, dx)`. При обходе CCW «влево» от направления ребра — это **внутрь** многоугольника. Поэтому `N` указывает внутрь.

#### Что значит `den = dot(N, D)`?

Это скалярная проекция вектора `D` (куда «летит» отрезок) на внутреннюю нормаль ребра:

```
den > 0  →  D и N сонаправлены  →  отрезок движется ВНУТРЬ полуплоскости  →  это точка ВХОДА (PE)
den < 0  →  D и N противоположны  →  отрезок движется НАРУЖУ  →  это точка ВЫХОДА (PL)
den = 0  →  D перпендикулярен N, то есть параллелен ребру  →  пересечения нет
```

#### Откуда формула `t = dot(N, A − P1) / den`?

Точка отрезка `P(t) = P1 + t·D`. Условие «точка лежит на плоскости ребра»:

```
dot(N, P(t) − A) = 0
dot(N, P1 + t·D − A) = 0
dot(N, P1 − A) + t·dot(N, D) = 0
t = −dot(N, P1 − A) / den = dot(N, A − P1) / den
```

#### Почему `tE = max(...)` и `tL = min(...)`?

Видимый отрезок должен одновременно лежать **внутри всех** полуплоскостей. Каждое ребро задаёт ограничение на допустимый интервал `t`:

- PE-пересечение говорит: «до параметра `t` мы снаружи этого ребра — значит, видимая часть начинается не раньше `t`». Чтобы быть внутри всех рёбер сразу, берём **максимум** из всех таких границ.
- PL-пересечение говорит: «после параметра `t` мы снаружи этого ребра — значит, видимая часть кончается не позже `t`». Берём **минимум**.

Если в итоге `tE > tL` — «войти» нужно уже после «выхода», интервал пустой, отрезок не пересекает окно.

#### Параллельный случай (`den == 0`)

Отрезок идёт параллельно ребру. Тогда он либо целиком в полуплоскости этого ребра, либо целиком снаружи. Проверяем: `dot(N, A − P1)` — это расстояние (со знаком) от P1 до прямой ребра. Если оно `< 0`, P1 снаружи, и раз D параллелен ребру, весь отрезок снаружи → `return None`.

#### Финальные точки Q₁ и Q₂

```python
Q1 = (p1[0]+tE*D[0], p1[1]+tE*D[1])   # P1 + tE · D
Q2 = (p1[0]+tL*D[0], p1[1]+tL*D[1])   # P1 + tL · D
```

Подставляем найденные параметры обратно в параметрическое уравнение отрезка.

### `cohen_sutherland(p1, p2)` — алгоритм Коэн–Сазерленд

```python
def cohen_sutherland(p1, p2):
    def code(x, y):              # 4-битный код региона точки
        c = 0
        if x < xmin: c |= 1     # бит 0: LEFT
        elif x > xmax: c |= 2   # бит 1: RIGHT
        if y < ymin: c |= 4     # бит 2: BOTTOM
        elif y > ymax: c |= 8   # бит 3: TOP
        return c
    x1, y1, x2, y2 = float(p1[0]), float(p1[1]), float(p2[0]), float(p2[1])
    c1, c2 = code(x1, y1), code(x2, y2)
    while True:
        if not (c1 | c2): return (x1, y1), (x2, y2)  # оба внутри
        if c1 & c2: return None                        # оба снаружи с одной стороны
        co = c1 or c2                                  # код наружного конца
        if   co & 8: x = x1+(x2-x1)*(ymax-y1)/(y2-y1); y = ymax   # пересечение TOP
        elif co & 4: x = x1+(x2-x1)*(ymin-y1)/(y2-y1); y = ymin   # пересечение BOTTOM
        elif co & 2: y = y1+(y2-y1)*(xmax-x1)/(x2-x1); x = xmax   # пересечение RIGHT
        else:        y = y1+(y2-y1)*(xmin-x1)/(x2-x1); x = xmin   # пересечение LEFT
        if co == c1: x1, y1 = x, y; c1 = code(x1, y1)  # заменяем первый конец
        else:        x2, y2 = x, y; c2 = code(x2, y2)  # заменяем второй конец
```

#### Переменные по одной

| Переменная | Что хранит | Начальное значение | Как меняется |
|------------|-----------|-------------------|-------------|
| `x1, y1` | Координаты первого конца (обновляются по ходу) | `float(p1)` | Заменяются точкой пересечения, когда `co == c1` |
| `x2, y2` | Координаты второго конца | `float(p2)` | Заменяются точкой пересечения, когда `co == c2` |
| `c1` | 4-битный код региона первого конца | `code(x1, y1)` | Пересчитывается после замены `(x1, y1)` |
| `c2` | 4-битный код региона второго конца | `code(x2, y2)` | Пересчитывается после замены `(x2, y2)` |
| `co` | Код того конца, который сейчас снаружи | — | Берётся как `c1 or c2` в каждой итерации |
| `x, y` | Координаты новой точки пересечения с границей | — | Вычисляются по формуле параметрической прямой |

#### Как читать 4-битный код

```
Пример: точка (-200, -150)
  xmin=-120, ymin=-100
  x < xmin  →  бит 0 = 1  (LEFT)
  y < ymin  →  бит 2 = 1  (BOTTOM)
  итого: c = 0b0101 = 5
```

Код внутренней точки = `0b0000 = 0`. Код точки, которая одновременно левее и выше = `0b1001 = 9`.

#### Почему `c1 | c2 == 0` → принятие, а `c1 & c2 != 0` → отклонение?

- `c1 | c2 == 0`: побитовое ИЛИ нулевое только если оба кода нулевые → оба конца внутри.
- `c1 & c2 != 0`: есть бит, который ненулевой у обоих → оба конца по одну сторону одной из четырёх граней → отрезок гарантированно не пересекает окно. Например, если оба `c & 8 != 0` — оба выше `ymax`.

#### Откуда формулы пересечения?

Параметрическое уравнение прямой: `P(t) = P1 + t·(P2 − P1)`. При пересечении с `y = ymax`:

```
y1 + t·(y2 − y1) = ymax
t = (ymax − y1) / (y2 − y1)
x = x1 + t·(x2 − x1) = x1 + (x2−x1)·(ymax−y1)/(y2−y1)
```

Код записывает это в одну строку без промежуточной переменной `t`.

#### Почему `co = c1 or c2`?

`or` в Python возвращает первый **истинный** операнд. Если `c1 != 0` — `co = c1`; иначе `co = c2`. Это выбирает **любой** конец, лежащий снаружи. Алгоритм «срезает» его по ближайшей нарушенной границе и повторяет цикл.

### `midpoint_clip(p1, p2, iters=52)` — метод средней точки

```python
def midpoint_clip(p1, p2, iters=52):
    def inside(x, y):           # True, если точка внутри прямоугольника
        return xmin <= x <= xmax and ymin <= y <= ymax
    def code(x, y):             # тот же 4-битный код, что в Коэн–Сазерленд
        ...
    x1, y1, x2, y2 = float(p1[0]), float(p1[1]), float(p2[0]), float(p2[1])
    in1, in2 = inside(x1, y1), inside(x2, y2)   # флаги: внутри ли каждый конец
    if in1 and in2: return (x1, y1), (x2, y2)   # оба внутри — сразу возвращаем
    if code(x1, y1) & code(x2, y2): return None  # оба снаружи с одной стороны

    def bisect(ox, oy, ix, iy):  # ox,oy — снаружи;  ix,iy — внутри
        for _ in range(iters):
            mx, my = (ox+ix)/2, (oy+iy)/2  # средняя точка
            if inside(mx, my): ix, iy = mx, my  # средняя внутри → сдвигаем внутр. конец
            else:               ox, oy = mx, my  # средняя снаружи → сдвигаем внешн. конец
        return (ox+ix)/2, (oy+iy)/2              # граничная точка между ox и ix

    nx1, ny1 = bisect(x1, y1, x2, y2) if not in1 else (x1, y1)  # граница со стороны P1
    nx2, ny2 = bisect(x2, y2, x1, y1) if not in2 else (x2, y2)  # граница со стороны P2
    return (nx1, ny1), (nx2, ny2)
```

#### Переменные по одной

| Переменная | Что хранит | Начальное значение | Как меняется |
|------------|-----------|-------------------|-------------|
| `in1, in2` | Булевы флаги: находится ли каждый из концов внутри прямоугольника | `inside(x1,y1)`, `inside(x2,y2)` | Не меняются — вычисляются один раз |
| `ox, oy` | **Внешний** конец текущего бинарного поиска | Конец отрезка, лежащий снаружи | Заменяется средней точкой, если средняя снаружи |
| `ix, iy` | **Внутренний** конец текущего бинарного поиска | Конец отрезка, лежащий внутри | Заменяется средней точкой, если средняя внутри |
| `mx, my` | Средняя точка текущего интервала `[outer, inner]` | — | Пересчитывается в каждой итерации |
| `nx1, ny1` | Граничная точка со стороны P1 (≈ Q₁) | — | Результат `bisect(P1, P2)` |
| `nx2, ny2` | Граничная точка со стороны P2 (≈ Q₂) | — | Результат `bisect(P2, P1)` |

#### Как работает бисекция — пошагово

Пусть P1 снаружи, P2 внутри. Вызываем `bisect(P1, P2)`:

```
Итерация 0: ox=P1(снаружи),  ix=P2(внутри),  mx = середина P1P2
  mx внутри  →  ix = mx   (внутренний конец сдвигается к P1)
Итерация 1: ox=P1(снаружи),  ix=mx_0(внутри), mx = новая середина
  mx снаружи →  ox = mx   (внешний конец сдвигается к P2)
...
После 52 итераций ox и ix находятся в пределах машинной точности друг от друга.
Граничная точка ≈ (ox + ix) / 2.
```

Инвариант на каждой итерации: `ox` всегда снаружи, `ix` всегда внутри. Граница зажата между ними, и с каждой итерацией интервал уменьшается вдвое.

#### Почему `iters = 52`?

`float64` имеет 52 бита мантиссы. После 52 делений пополам отрезок `[ox, ix]` сокращается до `2⁻⁵² ≈ 2.2 × 10⁻¹⁶` от исходной длины — это предел точности `float64`. Дальнейшие итерации не дадут ничего нового.

#### Почему вызов дважды с переставленными аргументами?

```python
bisect(x1, y1, x2, y2)  # P1-снаружи, P2-внутри → ищем Q₁
bisect(x2, y2, x1, y1)  # P2-снаружи, P1-внутри → ищем Q₂
```

Каждый вызов ищет **одну** граничную точку: ту, которая ближе к наружному концу. Аргументы меняются местами, чтобы `ox, oy` всегда был наружным, а `ix, iy` — внутренним. Если конец уже внутри — бисекция не нужна, возвращается сам конец.

### Вспомогательные функции рисования

```python
def seg(ax, a, b, **kw):
    ax.plot([a[0], b[0]], [a[1], b[1]], **kw)

def mark(ax, p, label='', **kw):
    ax.scatter(p[0], p[1], zorder=5, **kw)
    if label:
        ax.annotate(label, p, xytext=(6, 6), textcoords='offset points', ...)

def extend_line(ax, a, b, **kw):
    xl, yl = ax.get_xlim(), ax.get_ylim()
    dx, dy = b[0]-a[0], b[1]-a[1]
    ts = ([(xl[0]-a[0])/dx, (xl[1]-a[0])/dx] if dx else [-1e9, 1e9]) + \
         ([(yl[0]-a[1])/dy, (yl[1]-a[1])/dy] if dy else [-1e9, 1e9])
    ts.sort()
    t1, t2 = ts[1], ts[2]
    ax.plot([a[0]+t1*dx, a[0]+t2*dx], [a[1]+t1*dy, a[1]+t2*dy], **kw)
```

`seg` — тонкая обёртка над `ax.plot`, переводящая две точки в списки координат.

`mark` — рисует точку через `scatter` (поддерживает `marker`, `s`, `color`) и смещает подпись на 6 пикселей вправо–вверх. `zorder=5` гарантирует, что точки рисуются поверх линий.

`extend_line` — продлевает прямую, заданную двумя точками, до границ видимой области. Параметры `t` пересечения с четырьмя граничными прямыми (x = xl[0], x = xl[1], y = yl[0], y = yl[1]) собираются в список из четырёх значений. После сортировки два средних — `ts[1]` и `ts[2]` — это ровно те параметры, где прямая **входит** в прямоугольник видимой области и **выходит** из него. Крайние `ts[0]` и `ts[3]` соответствуют пересечениям вне видимой зоны и отбрасываются. Вырожденный случай (вертикальная или горизонтальная прямая, `dx == 0` или `dy == 0`) заменяется значениями `±1e9`, чтобы не возникало деления на ноль.
